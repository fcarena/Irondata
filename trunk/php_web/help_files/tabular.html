What is a Tabular Report?

A tabular report shows numeric data indexed by two or more properties.

Tabular Report
X Index
Y Index one two three four
one 1 2 3 4
two 5 6 7 8
three 9 10 11 12
four 13 14 15 16

Irondata looks at the data source and finds all records with identical values in the first column. From this set of data, Irondata finds all pieces of data with identical values for the second property.

Once Irondata has a this set of data, it will have a range of numerical values that are to be reported. From these values Irondata can then caluculate the sum, the average, the minimum, the maximum, the number of values, or the number of unique values. This calculation is called the Aggreggate.

This process is then repeated for each unique value for each of the properties. Aggreggates can then be organised into a grid, with one set of properties forming the X axis and one forming the Y axis.

Data Source

The data source is the numerical data that will be grouped and aggreggated to form the result of the report. This data must be numerical in nature.

Axis Relationship

Each axis column must be linked to the intersection column, for the data to be indexed. In some cases the intersection column and an axis column are linked simply by being two columns in the same table. This is called a self-referential join. In Many cases the intersection column and an axis column must be linked through an inner join, where one column references a column in another table.

Example: A simple self referential join from the axis column, at left, to the intersection column, at right, both in the same table.

Staff.Position ⇄ Staff.Name
The selected column is linked, via a self referential join, to the intersection column.

In many cases an inner join may not link the axis and intersection columns directly. The axis column may need to be self-referentially joined to an intermediate column, then inner joined to the intersection column, in another table.

Example: The axis and intersection columns, in different tables, are linked by both self referential join and inner joins.

Assignments.Location ⇄ Assignments.Staff Id references Staff.Staff Id ⇄ Staff.Name
The selected column is linked, via a self referential join, then via an inner join, then via another self referential join, to the intersection column.
